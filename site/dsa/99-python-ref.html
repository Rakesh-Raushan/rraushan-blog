Things that seem trivial but silently kill you in problems if you forget.

Iteration & Range
range() — edge cases that bite
range(start, stop, step) — stop is exclusive, no iteration when start ≥ stop (default step)
# These produce NOTHING — easy to forget under pressure
range(1, 1)    # start == stop → empty
range(1, 0)    # start > stop with default step +1 → empty
range(5, 2)    # same — empty, NOT [5,4,3]

# Reverse iteration — must pass step=-1
for i in range(5, 0, -1):   # → 5 4 3 2 1  (stop=0 excluded)
    ...
for i in range(4, -1, -1):  # → 4 3 2 1 0  (to include 0, stop=-1)
    ...

# Iterate indices of a list in reverse
for i in range(len(arr) - 1, -1, -1):
    ...
Classic bug: writing range(n, 0) thinking it goes backwards — it produces nothing. You need range(n, 0, -1).
enumerate() — index + value together
No manual counter variable needed
for i, ch in enumerate("hello"):       # i=0,ch='h' ...
    ...
for i, v in enumerate(arr, start=1):    # i starts at 1
    ...
Characters & Strings
ord() / chr() — character ↔ ASCII
Needed whenever a problem maps letters to indices or does arithmetic on characters
ord('a')           # → 97
ord('z') - ord('a')  # → 25  (26 letters, 0-indexed)
ord('A')           # → 65  (uppercase is separate block)

# Map letter to 0-25 index — extremely common
idx = ord(ch) - ord('a')

chr(97)             # → 'a'
chr(ord('a') + 3)   # → 'd'
Trie / frequency array problems: always index with ord(ch) - ord('a').
String checks — isdigit, isalpha, isalnum, islower
ch.isdigit()    # '5' → True,  'a' → False
ch.isalpha()    # 'a' → True,  '5' → False
ch.isalnum()    # 'a' → True,  '5' → True,  '!' → False
ch.islower()    # 'a' → True,  'A' → False
ch.isupper()    # 'A' → True
ch.lower()      # normalise case before comparing
Palindrome / two-pointer problems: always use isalnum() to skip non-alphanumeric characters before comparing.
Data Structures — built-in
set — use it whenever you need O(1) lookup
If a list is only being queried (not indexed), convert to set immediately
seen = set()
seen.add(x)
x in seen        # O(1) — not O(n) like a list

# Convert list → set for fast lookup
num_set = set(nums)
if target - x in num_set: ...   # two-sum style

# Set from string — unique chars
set("abracadabra")   # {'a','b','r','c','d'}
Rule: if the only operation on a collection is in, it should be a set, not a list.
collections.defaultdict — avoid KeyError boilerplate
from collections import defaultdict

freq = defaultdict(int)     # missing key → 0
freq[ch] += 1              # no need to check existence

graph = defaultdict(list)   # missing key → []
graph[u].append(v)
collections.Counter — instant frequency map
from collections import Counter

c = Counter("anagram")      # {'a':3,'n':1,'g':1,'r':1,'m':1}
c = Counter([1,1,2,3])     # Counter({1:2, 2:1, 3:1})
c.most_common(2)            # top 2 by frequency

# Anagram check in one line
Counter(s) == Counter(t)
heapq — min-heap (Python only has min-heap)
import heapq

h = []
heapq.heappush(h, 3)
heapq.heappop(h)             # returns smallest
h[0]                         # peek without pop

# Max-heap trick: negate values
heapq.heappush(h, -val)
max_val = -heapq.heappop(h)

# heapify in O(n)
heapq.heapify(arr)
No max-heap built-in. Negate values to simulate one. Don't forget to negate on both push and pop.
Control Flow
elif vs multiple if — short-circuit when only one branch should fire
Multiple ifs always evaluate all conditions. elif stops at first match.
# BAD — checks all three even after matching
if x == 1: handle_one()
if x == 2: handle_two()    # still evaluated
if x == 3: handle_three()  # still evaluated

# GOOD — stops at first match
if x == 1:   handle_one()
elif x == 2: handle_two()
elif x == 3: handle_three()
Use elif whenever exactly one branch should execute. The difference matters when conditions have side effects, or when you're inside a loop and want to avoid unintended double-processing.
for…else — runs if loop completed without break
for x in arr:
    if x == target:
        print("found")
        break
else:
    print("not found")   # only runs if no break hit
Cleaner than a boolean flag for "did we find it" patterns.
swap — one line, no temp variable
Python tuple unpacking makes swap a single expression
# The Python way — clean, no temp needed
a, b = b, a

# In-place swap in an array
arr[i], arr[j] = arr[j], arr[i]
Right-hand side is evaluated fully before any assignment happens, so no temp variable is ever needed. Use this everywhere — it's not just style, it's fewer variables to get wrong.
max() / min() for updates — skip the if statement
Cleaner and less error-prone than conditional assignment
# BAD — if statement for a simple update
if val > curr_max:
    curr_max = val

# GOOD — one line, intention is clear
curr_max = max(curr_max, val)
curr_min = min(curr_min, val)

# Common pattern: running window max/min
for num in nums:
    curr_max = max(curr_max, num)
    curr_min = min(curr_min, num)

# Also works with expressions
best = max(best, left + right)
result = min(result, cost[i] + 1)
Especially useful in DP — dp[i] = max(dp[i-1], ...) reads exactly like the recurrence relation, so the code maps directly to your reasoning.
Numbers & Math
Integer division, modulo, infinity
7 // 2          # → 3  (floor division)
-7 // 2         # → -4  (floors toward -∞, not 0 — watch this)
7 % 3           # → 1
-1 % 3          # → 2   (Python mod is always non-negative)

# Infinity — useful for "min so far" initialisation
float('inf')    # use as initial max/min sentinel
float('-inf')
Negative floor division: -7 // 2 is -4 in Python, not -3. If you want truncation toward zero, use int(-7 / 2) → -3.
Useful builtins for arrays
min(a, b)           # two values
min(arr)            # entire iterable
max(arr, key=abs)  # with key function
sum(arr)
sorted(arr)                         # new list
sorted(arr, key=lambda x: x[1])   # sort by 2nd element
arr.sort(reverse=True)             # in-place
zip(a, b)                           # pair-wise iteration
any(x > 0 for x in arr)
all(x > 0 for x in arr)
Two-pointer & Sliding Window patterns
Initialising 2D arrays correctly
# WRONG — all rows share the same list object
dp = [[0] * n] * m

# CORRECT
dp = [[0] * n for _ in range(m)]
Multiplying a list of lists creates copies of the reference, not independent rows. Always use a list comprehension for 2D initialisation.
